# Документация по скетчу шлюза

Файл: `sketch_doorhan_gsm_v_4_8_work/sketch_doorhan_gsm_v_4_8_work.ino`

## Назначение

Скетч реализует GSM-шлюз для ворот на Arduino + SIM800L:
- принимает входящие вызовы и открывает ворота по whitelist-номерам;
- поддерживает 2 группы доступа (`0` сразу, `1` с задержкой);
- синхронизирует номера/настройки с сервером по HTTP API;
- хранит список номеров во внешней SPI Flash;
- выполняет команды с сервера (`send_sms`, `send_ussd`, `close_gate`);
- отправляет события в логи на сервер.

## Основные блоки

- **Инициализация**: `setup()`, `initSIM800()`, `initFlash()`, `readHeader()`.
- **Главный цикл**: `loop()`:
  - чтение URC модема;
  - обработка звонков/SMS;
  - пошаговая синхронизация;
  - отложенное открытие для группы 1;
  - периодический sync и обработка серверных команд.
- **Синхронизация**:
  - `syncWithServer()` — старт режима `delta` или `full`;
  - `continueNumberDownload()` — по странице за проход;
  - `downloadDeltaRemovedPage()`, `downloadDeltaUpsertPage()`, `downloadPage()`;
  - `completeNumberDownload()` — запись хеша и времени sync в header.
- **SPI Flash**:
  - чтение/запись/стирание: `flashReadBytes()`, `flashWriteBytes()`, `flashSectorErase()`;
  - таблица номеров: `readRecord()`, `writeRecord()`, `findPhoneIndex()`, `findNumber()`;
  - безопасная пересборка при patch существующей записи: `rebuildPhoneTablePatchRow()`.
- **Команды сервера**: `checkServerCommands()`.
- **Логирование**: `sendLog_P()`, `sendLog_Str()`.

## Формат хранения номеров во Flash

- Запись: `RECORD_SIZE = 17` байт:
  - 16 байт номер (нормализованная строка),
  - 1 байт группа (`0`/`1`).
- Две области:
  - основная таблица `FLASH_DATA_BASE`,
  - теневая таблица `FLASH_SHADOW_BASE` (для безопасной пересборки).

## Почему есть теневая таблица

NOR Flash пишет биты только `1 -> 0`, без стирания нельзя надёжно изменить существующий байт на `0 -> 1`.
Поэтому при изменении существующей записи:
1. пересборка в тень;
2. стирание основной области;
3. копирование обратно.

Это защищает список от «перезатирания всего» при изменении одной записи.

## Синхронизация с сервером

- `get_hash`:
  - получает `hash`, `server_time`, настройки и актуальный токен;
  - сравнивает с локальным `header.hash`.
- При различии хеша:
  - сначала попытка `delta` (`removed` + `upserts`);
  - fallback на `full` при сигнале `full_sync_required` или внутренней ошибке.
- В `delta` режим не блокирует loop: по одной странице за проход.

## Логика звонков

- Из URC `+CLIP` парсится номер.
- Номер нормализуется (`normalizePhone`) и ищется в таблице.
- Группа:
  - `0`: мгновенное `openGate()`;
  - `1`: старт таймера `delayActive` на `relay_delay_group1_ms`.
- Во время работы реле UART модема продолжает опрашиваться (`pumpSim800ClipWhile()`).

## Логика SMS/USSD/команд

- `send_sms`: отправка через `AT+CMGS`, затем `command_ack`.
- `send_ussd`: `AT+CUSD=1,"code",15`, чтение `+CUSD` в `readUssdResponseInto()`, затем `command_ack`.
- `close_gate`: закрытие реле + лог + `command_ack`.

## Важные ограничения

- Буферы и парсинг JSON максимально упрощены для SRAM AVR.
- Нет ArduinoJson (целенаправленно).
- HTTP вызовы должны быть короткими, иначе возможны задержки в реакции на входящие события.
